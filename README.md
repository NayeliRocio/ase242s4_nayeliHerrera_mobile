## 📌 Definición del Problema

El restaurante Altavista Rooftop presenta dificultades en la gestión de reservas, ya que actualmente se realizan de forma manual o mediante mensajes, lo que genera desorganización, duplicidad de reservas y posibles errores en la atención al cliente.

Se propone el desarrollo de una aplicación móvil que permita gestionar las reservas de manera eficiente y segura.

## 👥 Usuarios del Sistema

* Administrador
* Cliente

## 🎯 Objetivos

* Automatizar el proceso de reservas
* Mejorar la organización de mesas
* Evitar errores en reservas
* Mejorar la experiencia del cliente
* Permitir acceso desde dispositivos móviles

## ⚙️ Requerimientos Funcionales

* RF01: Registro de clientes
* RF02: Inicio de sesión
* RF03: Realizar reservas
* RF04: Visualizar disponibilidad de mesas
* RF05: Gestión de reservas (CRUD)
* RF06: Cancelación de reservas
* RF07: Listado de reservas por fecha

## 🔐 Requerimientos No Funcionales

* RNF01: Tiempo de respuesta menor a 3 segundos
* RNF02: Interfaz amigable
* RNF03: Seguridad de la información
* RNF04: Compatibilidad con Android
* RNF05: Disponibilidad 24/7

## 📖 Historias de Usuario

* HU01: Como cliente, quiero registrarme para poder reservar.
* HU02: Como cliente, quiero iniciar sesión para acceder al sistema.
* HU03: Como cliente, quiero ver mesas disponibles.
* HU04: Como cliente, quiero reservar una mesa.
* HU05: Como cliente, quiero cancelar una reserva.
* HU06: Como administrador, quiero gestionar reservas.
